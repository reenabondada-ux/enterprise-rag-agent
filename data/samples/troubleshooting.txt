Troubleshooting Guide

Common Issues and Solutions:

Issue: "extension 'vector' is not available"
Solution: The pgvector extension is not installed on your PostgreSQL server. If using Docker, ensure you're using the ankane/pgvector image or postgres image with pgvector installed. For local Postgres, install via package manager (brew install pgvector on macOS).

Issue: "OPENAI_API_KEY is missing"
Solution: Set the OPENAI_API_KEY environment variable in your .env file or export it in your shell. The embedding functions require a valid OpenAI API key to generate embeddings.

Issue: "ModuleNotFoundError: No module named 'services'"
Solution: Run commands from the repository root with PYTHONPATH=. to ensure Python can find the services package. Example: PYTHONPATH=. python services/ingestor/ingest.py

Issue: "operator does not exist: vector <-> bytea"
Solution: You're passing the wrong type to the vector similarity operator. Ensure you're passing a list of floats and casting it to vector in SQL: %s::vector

Issue: Database connection timeout
Solution: Ensure PostgreSQL is running (docker compose ps) and the DATABASE_URL in your .env matches your database configuration. Check firewall rules if connecting to remote database.

Issue: Slow embedding generation
Solution: Embeddings are generated via OpenAI API calls which can be slow. For production, consider:
- Caching embeddings in Redis
- Batching embedding requests
- Using a faster embedding model
- Pre-computing embeddings offline

Issue: Poor retrieval quality
Solution: Retrieval quality depends on several factors:
- Chunk size: Too large or too small chunks hurt relevance
- Overlap: Ensure chunks overlap to preserve context
- Embedding model: Consider upgrading to larger embedding models
- Query formulation: Rephrase queries for better matching
- Re-ranking: Add a cross-encoder re-ranking step
